用户注册处理流程
====

1. 用户请求页面
----
    1.1 通过captchaSrv获取一个captcha对象, 获取token与url,以及有验证的Valid方法
    1.2 在页面上添加隐藏字段表示要提交的captcha的token, 在页面上显示验证码图片

2. 用户刷新验证码
----
    2.1 一个独立的js请求, 带上隐藏表单里的captcha token
    2.2 根据captcha token将原来保存的beego captcha对象删除
    2.3 返回新的captcha token 与url
    2.3 (TODO)根据xsrf token添加刷新次数判断,比如在一分钟里不能刷新120次, 否则此接口是一个漏洞


3. 用户提交请求
----
    3.1 验证码判断
    3.2 其它字段判断
    3.3 验证手机号或者邮箱是否被使用

4. 邮箱注册
----
    4.1 发送email, 生成确证地址

5. 手机注册
----

6. 保存用户资料
----
    4.1 插入数据库
    4.2 生成用户session数据
    4.3 成功跳转, 失败将数据写进flash里,302重定向显示错误信息
