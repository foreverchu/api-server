关于错误码的改进
====

目前错误码规范:
----
从-1到-1003

问题:

1. 不直观, 只能通过文档对错误查询, 才知道其确切含义
2. 添加错误时不规范, 通过递减数字添加新的错误


解决方案:
----

> 模块名 + HTTP status code + bussiness logic code

错误码 | 模块名 | http status code | code | desc
---- | ----| ---- | ---- | ---- 
USR200001 | USR | 200 | 001 | 用户注册成功
USR200002 | USR | 200 | 002 | 获取验证码成功
USR302001 | USR | 301 | 001 | 资源已经被重定向
USR400001 | USR | 400 | 001 | 参数错误, 手码号码不正确
USR400002 | USR | 400 | 002 | 参数错误, 手码验证号码不正确
USR403001 | USR | 403 | 001 | 请求无权限
USR403002 | USR | 403 | 002 | 请求失败, token 过期
USR403003 | USR | 403 | 003 | 请求失败, token 验证失败
USR500001 | USR | 500 | 001 | 服务器提出一个问题, 服务器出错了
